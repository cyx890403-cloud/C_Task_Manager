# 伺服器任務排程管理系統 (Server Task Scheduler)

## 📝 專題名稱與功能介紹
本專案為一個模擬伺服器後端處理機制的排程系統。透過 C 語言實作高效率的佇列（Queue）管理，並具備以下核心功能：
* **優先權排程**：支援緊急任務插隊機制，模擬真實伺服器的急件處理。
* **自動日誌系統**：即時紀錄任務執行狀況至 `server.log`。
* **狀態持久化**：關機自動存檔 `tasks.txt`，開機自動還原待辦任務。
* **TUI 儀表板**：彩色終端機介面與進度條動畫，提供優質的使用者體驗。

---

## 🛠 使用技術 (Pointer, Malloc, 資料結構)
本專案深度應用 C 語言核心技術，完全符合軟體工程規範：
* **指標 (Pointers)**：透過指標精確維護 Linked List 節點的連結與插隊邏輯。
* **動態記憶體 (Malloc/Free)**：
  - 使用 `malloc` 動態配置每個任務節點。
  - 在任務執行完畢 (Dequeue) 時立即 `free` 記憶體，確保 **0% Memory Leak**。
* **資料結構 (Queue)**：實作具備優先權權重的單向佇列 (Priority Queue)。
* **檔案處理 (File I/O)**：
  - 使用 `fopen`, `fprintf` 實作資料存檔。
  - 使用 `fgets`, `sscanf` 解析外部文字檔資料。

## 📂 系統架構與執行方式 (How to run)
### 專案結構
本專案採用模組化設計，將宣告與實作分離：
* `queue.h`：定義 `struct` 結構與函式介面宣告。
* `queue.c`：核心演算法（Enqueue/Dequeue）、檔案讀寫與日誌邏輯。
* `main.c`：負責選單 UI 呈現與使用者互動。

### 編譯與執行
請在終端機輸入以下指令：
```bash
# 編譯指令
gcc main.c queue.c -o task_manager

# 執行指令
./task_manager.exe
```
